# Constraint Files

For this project, only keep XDC files here if they are genuinely useful design
inputs.

## Keep

Keep an XDC if it was manually authored or manually edited and it contains
project-specific constraints such as:

- external pin assignments
- board I/O mapping
- user-defined clocks
- false-path or multicycle constraints you added intentionally
- any custom timing exceptions needed by the final design

## Usually Do Not Keep

The following are usually auto-generated by Vivado / IP Integrator and are not
good canonical repo inputs by themselves:

- `dont_touch.xdc`
- `*_ooc.xdc`
- `*_oclocks.xdc`
- `*_auto_*.xdc`
- IP-specific generated XDCs that can be recreated from the block design

Examples from this project snapshot that are usually generated rather than
hand-authored:

- `design_1_auto_pc_1_ooc.xdc`
- `design_1_auto_ds_0_clocks.xdc`
- `design_1_auto_ds_0_ooc.xdc`
- `design_1_xbar_0_ooc.xdc`
- `design_1_rst_ps7_0_50M_0_*.xdc`
- `design_1_axi_dma_0_0_*.xdc`
- `dont_touch.xdc`

## Practical Recommendation

If your design was mainly created through Vivado block design and you did not
manually add constraints, it is better to:

- keep the `.bit` and `.hwh` in `../export/`
- keep Tcl / block-design source in `../source/`
- keep reports/screenshots in `../reports/` and `../images/`
- leave this folder with only a note like this rather than dumping every
  generated XDC

If you are unsure whether an XDC was manually created, check whether you wrote
or edited it yourself. If not, it is probably safe to leave it out.
